## Why This Topic

RL is the backbone of two later topics: DPO/PPO (how LLMs are aligned to
human preferences) and Agentic AI (how an LLM decides which tool to call
next). Both are structurally the same loop you'll learn here. Get the loop
right once and the rest of the week is much easier to follow.

## What You Will Learn

**The RL loop:**
- Agent, environment, state, action, reward — and how they fit into a
  Markov Decision Process (MDP)
- Discounted return, and why we discount future rewards at all

**Two families of methods:**
- Value-based: Q-learning, and how Deep Q-Networks (DQN) scale it up
- Policy-based: the policy gradient theorem, actor-critic, and PPO

**Two axes every RL method sits on:**
- Exploration vs. exploitation
- On-policy vs. off-policy learning

## Concepts in Depth

### 🔄 The Core Loop

An **agent** interacts with an **environment** over discrete time steps. At
each step it observes a **state** `s`, takes an **action** `a`, and
receives a **reward** `r` plus a new state `s'`. The goal is to learn a
**policy** `π(a|s)` that maximizes expected cumulative discounted reward:

```
G_t = r_t + γ·r_{t+1} + γ²·r_{t+2} + ...
```

`γ ∈ [0,1)` trades off immediate vs. future reward and keeps infinite sums
finite. This is formalized as an **MDP**: `(S, A, P, R, γ)`, where the
Markov property says the next state depends only on the current state and
action — not the full history.

### 📊 Value-Based Methods

**Q-learning** learns `Q(s,a)`: the expected return of taking action `a` in
state `s` and acting optimally afterward. It updates via the Bellman
equation:
```
Q(s,a) ← Q(s,a) + α[r + γ·max_a' Q(s',a') − Q(s,a)]
```

**Deep Q-Networks (DQN)** replace the Q-table with a neural network for
large/continuous state spaces. Two tricks make this stable: **experience
replay** (breaks correlation between consecutive samples) and a **target
network** (stabilizes the bootstrapped target).

### 📈 Policy-Based Methods

Instead of deriving actions from a value function, directly parameterize
`π_θ(a|s)` and push its parameters toward higher expected reward — the
**policy gradient theorem**:
```
∇J(θ) = E[∇log π_θ(a|s) · G_t]
```
Intuition: increase the probability of actions that led to high return,
decrease it for actions that led to low return.

**Actor-Critic** combines both families: an "actor" (policy) picks actions,
a "critic" (value function) reduces variance by replacing the raw return
with an advantage `A(s,a) = Q(s,a) − V(s)`.

**PPO (Proximal Policy Optimization)** is the actor-critic variant that
dominates in practice — including RLHF (see [Topic 2](../02-DPO-PPO/README.md)).
It clips the policy update so a single batch of data can't push the policy
too far, which is what makes training stable enough to run on something as
high-dimensional as language generation.

### ⚖️ Two Axes to Keep Straight

- **Exploration vs. exploitation** — try new actions vs. use what already
  works. Common strategies: ε-greedy, entropy bonuses, optimism under
  uncertainty.
- **On-policy vs. off-policy** — on-policy methods (PPO, vanilla policy
  gradient) can only learn from data generated by the *current* policy;
  off-policy methods (Q-learning, DQN) can learn from any data, including
  a replay buffer of old experience. On-policy is simpler and more stable;
  off-policy is more sample-efficient.
